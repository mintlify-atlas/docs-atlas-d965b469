All Dify configuration is managed through environment variables. In Docker Compose deployments these live in the `docker/.env` file. Copy `docker/.env.example` to `docker/.env` to get started.

<Note>
Variables marked **Required** must be set to a non-default value for a secure, functional production deployment. Variables marked **Optional** have sensible defaults that work out of the box.
</Note>

## Core / common settings

| Variable | Default | Required | Description |
|---|---|---|---|
| `SECRET_KEY` | `sk-9f73s3lj...` | **Required** | Signs session cookies and encrypts sensitive DB data. Generate with `openssl rand -base64 42`. |
| `INIT_PASSWORD` | _(empty)_ | Optional | Pre-set the initial admin password. Max 30 characters. |
| `DEPLOY_ENV` | `PRODUCTION` | Optional | `PRODUCTION` or `TESTING`. Testing shows a color label in the UI. |
| `CONSOLE_API_URL` | _(empty)_ | Optional | Backend URL for the console API. Defaults to same domain. |
| `CONSOLE_WEB_URL` | _(empty)_ | Optional | Frontend URL for the console. Used for CORS and auth redirects. |
| `SERVICE_API_URL` | _(empty)_ | Optional | URL displayed as the Service API base URL in the UI. |
| `APP_API_URL` | _(empty)_ | Optional | Backend URL for WebApp API. |
| `APP_WEB_URL` | _(empty)_ | Optional | WebApp frontend URL. |
| `FILES_URL` | _(empty)_ | Optional | URL prefix for file downloads and previews. Required for multimodal file access. |
| `OPENAI_API_BASE` | `https://api.openai.com/v1` | Optional | Override the OpenAI base URL (for proxies or local OpenAI-compatible servers). |
| `CHECK_UPDATE_URL` | `https://updates.dify.ai` | Optional | Version check endpoint. Set to empty to disable. |
