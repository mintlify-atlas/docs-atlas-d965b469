<Note>
Ollama is the recommended way to use local models with Dify. After starting Ollama, add it as a provider in **Settings → Model Providers** and set the base URL to `http://host.docker.internal:11434` (Docker) or `http://localhost:11434` (native).
</Note>

## Model type support by provider

Dify supports five model types. Not every provider supports all types.

| Type | Description | Example providers |
|---|---|---|
| **LLM** | Text generation and chat | OpenAI, Anthropic, Google, Ollama |
| **Text embedding** | Convert text to vector representations for RAG | OpenAI (`text-embedding-3`), Cohere (`embed-v3`), Ollama (`nomic-embed-text`) |
| **Reranking** | Re-score retrieved documents for relevance | Cohere (`rerank-v3`), Jina AI, Voyage AI |
| **Text-to-speech (TTS)** | Generate spoken audio from text | OpenAI (`tts-1`), Azure OpenAI |
| **Speech-to-text (STT)** | Transcribe audio to text | OpenAI (`whisper-1`), Azure OpenAI |

<Tip>
You can configure multiple providers simultaneously. Dify lets you assign different providers for different model types — for example, OpenAI for LLM and Cohere for reranking.
</Tip>

## Using Ollama for local models

Ollama lets you run large language models entirely on your own hardware with no API key required.

<Steps>
  <Step title="Install and start Ollama">
    Download from [ollama.com](https://ollama.com) and pull a model:

    ```bash
    ollama pull llama3
    ollama pull nomic-embed-text
    ```
  </Step>
  <Step title="Add Ollama as a provider in Dify">
    Go to **Settings → Model Providers → Ollama**. Set the base URL:

    - Docker deployment: `http://host.docker.internal:11434`
    - Native / same host: `http://localhost:11434`
  </Step>
  <Step title="Add individual models">
    In the Ollama provider settings, click **Add Model** and enter the model name exactly as shown in `ollama list` (e.g., `llama3`, `nomic-embed-text`).
  </Step>
  <Step title="Set as system model">
    Go to **Settings → Model Providers → System Model Settings** and assign your Ollama model as the default LLM or embedding model.
  </Step>
</Steps>
