This guide walks you through running Dify on your own machine using Docker Compose — the fastest path to a working local instance.

## Prerequisites

<Note>
Make sure your machine meets these minimum requirements before proceeding.
</Note>

| Requirement | Minimum |
|---|---|
| CPU | 2 cores |
| RAM | 4 GiB |
| Disk | 10 GiB free |
| Docker | 20.10.0+ |
| Docker Compose | v2.0.0+ |

Install Docker and Docker Compose from [docs.docker.com](https://docs.docker.com/get-docker/).

## Deploy with Docker Compose

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/langgenius/dify.git
    cd dify
    ```
  </Step>
  <Step title="Configure environment variables">
    ```bash
    cd docker
    cp .env.example .env
    ```

    Open `.env` in your editor. At minimum, change these values for a production-like setup:

    ```bash .env
    # Generate a strong random secret key
    SECRET_KEY=your-secret-key-here

    # Change default passwords
    DB_PASSWORD=difyai123456
    REDIS_PASSWORD=difyai123456
    ```

    <Warning>
    The default `SECRET_KEY` in `.env.example` is a placeholder. Always set a strong unique value — this key is used to sign session tokens and encrypt sensitive data.
    </Warning>
  </Step>
  <Step title="Start all services">
    ```bash
    docker compose up -d
    ```

    This pulls images and starts all containers. The first run may take a few minutes while images download.

    You can follow the logs:
    ```bash
    docker compose logs -f
    ```
  </Step>
  <Step title="Check service health">
    Wait until all containers are healthy:
    ```bash
    docker compose ps
    ```

    You should see these services running:

    | Service | Description |
    |---|---|
    | `api` | Flask backend API |
    | `worker` | Celery task worker |
    | `web` | Next.js frontend |
    | `db` | PostgreSQL database |
    | `redis` | Redis cache/broker |
    | `nginx` | Reverse proxy |
    | `weaviate` | Vector database (default) |
    | `sandbox` | Code execution sandbox |
  </Step>
  <Step title="Initialize Dify">
    Open your browser and navigate to:

    ```
    http://localhost/install
    ```

    Complete the admin setup form:
    1. Enter your admin email and password
    2. Set your workspace name
    3. Click **Set Up**

    You will be redirected to the Dify dashboard.
  </Step>
  <Step title="Create your first app">
    From the dashboard:
    1. Click **Create App** in the top-right corner
    2. Choose an app type (Chatbot, Agent, Workflow, or Text Generator)
    3. Enter a name and description
    4. Click **Create**

    You're now in the Prompt IDE where you can select a model and start building.
  </Step>
</Steps>

## Add a model provider

Before your app can generate responses, you need to configure at least one LLM provider.

<Steps>
  <Step title="Open model settings">
    Click your workspace avatar in the top-right → **Settings** → **Model Provider**
  </Step>
  <Step title="Choose a provider">
    Select a provider (e.g., OpenAI, Anthropic, Google) and click **Setup**.
  </Step>
  <Step title="Enter credentials">
    Paste your API key and click **Save**. Dify will validate the key before saving.
  </Step>
</Steps>

<Tip>
Don't have API keys? Use **Ollama** to run a local model for free. See [Models & Providers](/models/add-model) for instructions.
</Tip>

## Stop and restart

```bash
# Stop all services
docker compose down

# Restart
docker compose up -d
```

Data is persisted in Docker volumes — your apps and knowledge bases are safe across restarts.
